## Running the app

```bash
Esse exemplo segue uma API de Login e senha para testar meu sisteminha, no momento eu montei apenas um CRUD simples de cadastro de ferias, vou deixar ele no meu git e implementar a regra de negocio, mas o processo de CRUD apenas funciona com os passos de autenticar o cliente.

Nesse modelo utilizamos o padrao Nestjs documentado com o swagger.

*Utilizer o comando "Yarn" para instalar as dependencias e adicone as config no .env seguindo a pasta config -> env -> dev.env para melhor funcionamento.

*Pode rodar o sistema com o comando "yarn run start:dev"

Geralmente de padrao para testar o backend com o front/swagger, acesse o http://localhost:3000/docs#/

Ao acessar, lembre-se de realizar o login no Client - /client/authentication/authenticate e logo apos coletar o token e validar no Authorize. Seguindo esse processo, vai estar habilitado para realizar os processos no vacation.


# watch mode
